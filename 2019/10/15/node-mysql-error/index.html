Node.js 链接Mysql 报错及解决方案

Oct 15, 2019 | Node.js
条评论

错误信息：

Error: ER_NOT_SUPPORTED_AUTH_MODE: Client does not support authentication protocol requested by server; consider upgrading MySQL client

报错原因：

mysql8.0.4开始引入一个caching_sha2_password模块作为默认身份验证插件，数据库连接时验证身份的工作方式(handshake process)会与以往不同。
但以前版本的通过mysql_native_password 创建的账户仍然可以正常工作，只是验证这些账户时会切回mysql_native_password的工作方式，以此实现向下兼容。
nodejs版本尚未跟进caching_sha2_password的实现，所以需要通过命令来手动切换验证账号方式

解决方法：

ALTER USER ‘root’@’localhost’ IDENTIFIED WITH mysql_native_password BY ‘password’;
