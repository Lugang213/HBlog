Node.js 术语表

第 1 章

  1. ECMAScript 标准 ECMAScript 是由 Ecma 国际标准化的脚本语言规范。ECMAScript 有几个标准，本书关注的是 ECMAScript 2015（ECMAScript 第 6 版）。为确保他们的解释器与为其他实现而写的 JavaScript 兼容，JavaScript 实现者采用了 ECMAScript 标准。
  2. 异步 不一定按照出现的顺序执行的代码。在 Node.js 中，这个术语通常用来描述那些接受回调函数为参数的 API，因为这些回调函数是在将来的某一时点运行的。比如说，作为fs.readFile 参数的那个回调函数，会在文件读取完成后收到该文件中的内容。
  3. 语义版本 使用三个数字表明版本兼容性的惯用法：主版本号、次版本号、修订号，表示为 1.0.2（主版本号为 1，次版本号为 0，修订号为 2）。一个依赖于 1.0.2 版本的项目应该能跟 1.1.1 兼容，但不能跟 2.0.2 兼容。
  4. Promise Promise 对象是标准化的 ECMAScript 2015 API，用于表示现在、将来可获得，或永远都得不到的值。
  5. 非阻塞 I/O 阻塞的操作会让线程挂起，直到操作完成才会继续执行。Node 用的是非阻塞 I/O，即从文件或网络等资源读取数据时不会阻塞线程的执行。
  6. npm Node 的包管理器。用来安装存放在大型中心仓库上的包，以及管理 Node 项目中的依赖项。
  7. libuv Node 用的多平台异步 I/O 库。Julia 等语言也在用这个库。
  8. 核心模块 Node 自带的那些库。
  9. 箭头函数 简写的函数。即在将函数作为参数传给其他函数时，用 () => {} 而不是function () {} 这样的写法。如果函数只接受一个参数，那么括号可以忽略。
  10. 解构 ECMAScript 2015 引入了解构，允许将对象和数组分解为变量和常量。比如说，const { name } = { name: ‘Alex’ }的计算结果是创建一个名为 name，值为 Alex的常量。
  11. JSON（JavaScript 对象表示法） JSON 是一种轻量的数据交换格式，基于 JavaScript 的子集，易于阅读和编写。
  12. 抽象接口 没有具体实现的 API 的程序化描述。Node.js 中的流 API 就是抽象接口。
  13. 剩余参数 ECMAScript 2015 中的剩余参数语法允许我们将一个不定数量的参数表示为一个数组。比如要命名两个参数，但其余参数都放在数组中，可以表示为 function (a, b, …rest)。还可以跟解构配合使用来复制对象：const newObject = { …oldObject }。
  14. 事件 导致某个函数被调用的字符串。该函数被称为对象监听器。发出事件的是发射器。Node 中用来创建发射器的基类是 EventEmitter。
  15. 事件轮询 Node 的事件轮询等待外部事件，并将它们转化为回调函数的调用。其他系统采用类似的机制（消息派发器和运行轮询）快速将事件路由给相应的事件处理器。
  16. REPL（读取计算输出循环） 可以用来计算代码及查看结果的命令行界面。

第 2 章

  1. 闭包 JavaScript 函数能捕获在其封闭作用域内定义的变量。比如在函数 A 内定义一个函数 B，则 B 能访问 A 中定义的所有值。
  2. package.json Node 项目中的文件，用来定义项目名称、作者、版权许可和依赖项。所有 Node 程序和库中都应该有这样一个文件。
  3. 模块 Node 模块是包含 JavaScript 代码的文件。可以输出值（一般是函数和常量）供其他文件使用。
  4. 栈跟踪 截止到错误发生时所执行过的程序指令。
  5. 内容管理系统（CMS） Web 程序，用来编辑文本和图片，编辑好的内容将会显示在面向公众的网站上。
  6. 流程控制（或控制流程） 语句执行的顺序。因为 Node 是异步的，所以控制流很重要。JavaScript 中有很多种处理控制流的办法，包括回调、Promise、生成器、基本循环原语，以及遍历器。在 Node 中，流控制是指将异步任务的执行顺序分组的办法。
  7. 回调函数 已经被传给另一个函数，并且可能稍后会调用的函数。
  8. 回调嵌套 回调内还有回调。在把某个回调函数作为参数传给一个函数时，某些情况下有必要在这个回调函数内再定义一个回调函数。
  9. 全局作用域 因为作用域是指值可以访问的范围，所以全局作用域的值在程序中的任何地方都可以访问。
  10. CommonJS 模块规范 用于定义应该从当前 JavaScript 文件中输出什么的一种模块格式。
  11. 状态 程序中的所有变量在指定时间点上的值。
  12. 属性 JavaScript 对象是包含键值对的集合，这些键值对就是对象的属性。

第 3 章

  1. 表单编码 在向 Web 服务器发送 HTTP POST 请求时，会包含一个简单的表单 POST，表单中的内容会编码为请求主体。最常用的格式 application/x-www-form-urlencoded，类似于 URL 编码，会将不安全的 ASCII 字符替换为百分号。
  2. 模板 用于生成 HTML 的普通文本格式，可以包含嵌入数据和 JavaScript 代码，以便精简HTML 的语法。
  3. MIME（多用途互联网邮件扩展） 这是一个互联网标准，用于往电子邮件和多部分消息体中添加非文本数据，以便让电子邮件客户端可以显示 HTML、图片和非 ASCII 字符集中的文本。
  4. 对象-关系映射（ORM） 对程序员友好的数据结构（比如 JavaScript 对象）和数据库的数据结构（比如表和外键）两者之间的映射是用这样的库建立的。
  5. 套路化代码 经常复制并且可以自动生成的代码。
  6. 路由 给定的路由处理器需要处理的 URL 片段和 HTTP 动词。
  7. 路由处理器 用户定义的回调函数，在有 HTTP 请求发送到 Web 程序时运行。路由处理器一般会生成内容，这些内容可能是来自数据库的，或者是对数据库进行修改，然后生成响应消息。这些响应消息可能是用模板生成的，也有可能是用 JSON 之类的格式。
  8. 客户端包 经过预处理的 JavaScript 代码，通常来自多个源文件，经过最小化和压缩后发送到客户端。
  9. 静态资源文件 无须 Web 服务器做任何额外处理就可以作为响应消息的文件。通常包括图片、CSS 文件和客户端 JavaScript 文件。
  10. cURL 一个用来发送 HTTP 请求的命令行工具和程序库。经常用作调试工具，可以快速检查 Web 服务器对请求的响应。
  11. 数据库模型 跟使用数据库的原生语言相比，设计良好的数据模型会让程序员感觉跟数据库表或文档的交互更轻松。
  12. REST（表述性状态转移），RESTful API 无状态 Web API 使用一组 HTTP 预先定义好的操作。这些操作是基于 HTTP 动词的，最常用的是 GET、POST、PUT 和 DELETE。

第 4 章

  1. 源码映射 一个文件，浏览器中的调试器可以据此将转译后源码文件中的代码映射到原始文件中的对应行上。
  2. Webpack 加载器 转换或转译源码。
  3. Webpack 插件 修改构建进程本身的行为，不一定会改变输出文件。
  4. 方法链 在上一个执行的方法的返回值上运行一个方法。
  5. 流 高效的数据输入和（或）输出通道，文本或二进制数据都可以。Node 支持可读、可写和其他流，并且这些流可以用管道连接到一起。
  6. 构建系统 一套工具和配置文件，其所生成的 JavaScript 能在浏览器中运行更高效。
  7. 管道 将一个数据输出连到另一个输入上。在 UNIX 中，进程是用竖线符（|）连成管道的；在 Node 中，流是用方法链连成管道的。
  8. 剪毛器 检查源码格式的正确性。剪毛器可以按照一组剪毛规则对项目进行检查，从而强化指定的编程风格。
  9. 测试引擎 运行并整理单元测试结果的程序，一般可以同时处理多个文件。
  10. 转译 也称为源码到源码编译，JavaScript 转译器可以将一种 ECMAScript 转换成另外一种。最常用的是将更先进的 ES2015 转换成向后兼容的 ECMAScript 5，后者能用在更多浏览器上。还有 TypeScript，它是 JavaScript 的超集，也能转译为 ES5 或 ES2015。

第 5 章

  1. Web 框架 用来开发 Web 程序的一组函数库，可以用插件或中间件进行扩展。
  2. 模型-视图-控制器（MVC） 将软件分解为组件的设计模式。模型管理数据和逻辑，视图将数据转化成用户界面，控制器则将交互转化成对模型和视图的操作。
  3. 单页 Web 程序 一次性返回给浏览器的程序，不需要整页刷新。如果程序需要改变浏览器中的 URL，可以用 HTML5 的历史 API 让用户觉得 URL 已经变了，而浏览器已经从服务器上加载了新的页面。
  4. 同构 通过共享相同的代码而能够在客户端和服务器端运行的 JavaScript 程序。
  5. GET 参数 出现在问号之后的 URL 参数，分隔符是 & 符号。
  6. 关系型数据库 基于所存储的实体及其关系而形成的数据库结构。
  7. HTTP 动词 HTTP 方法（GET、 POST、 PUT、PATCH、 DELETE）表示应该在远程资源上执行的动作。
  8. 解耦 如果项目中的某个函数、类或模块可以轻松替换，或者用在其他项目中，那么它就是松散耦合的。
  9. 全栈框架 如果框架中所包含的功能既可以用于客户端，也可以用于服务器端代码，则说它是全栈框架。那通常意味着这个框架中有处理 HTTP 请求、请求路由、数据库建模和与浏览器中运行的代码进行通信等功能的函数库。
  10. 中间件 能够按顺序调用来修改 HTTP 请求和响应的函数。
  11. 数据库适配器 一些通用的数据库的函数库，可以用特定的适配器进行扩展，以实现特定数据库所需的功能。

第 6 章

  1. bcrypt 密码散列函数。由于这个函数能将任意数量的数据映射为固定大小的字符串，因此用户密码的明文经过处理后可以安全地存储在数据库中。
  2. 模板语言 轻量的标记语言，可以转换为 HTML，并能够从代码中注入值，循环遍历数组或对象。
  3. 密码盐 用来作为散列函数输入的随机数据，可以加大字典攻击的难度。
  4. 单线程 运行的程序（进程）可以有多个并发执行的线程。JavaScript 的模型是使用单线程，但在发生事件时，线程可以切换上下文运行不同的代码。浏览器中的事件是用户点击按钮之类的交互动作。在 Node 中，通常是 I/O 事件，比如网络操作或从硬盘中读取数据。
  5. 第三方中间件 不是由初始的 Web 库或框架的作者发布的中间件组件。
  6. 内容协商 HTTP 标准的一部分，用来处理相同 URI 上不同版本的文档。如果服务器支持内容协商，那么用户代理（浏览器）可以请求不同格式的数据。
  7. 响应对象 决定服务器将会如何响应某个 HTTP 请求的对象。包含响应主体（通常是一个 Web 页面）和消息头。
  8. CSS 预处理器 将 CSS 超集转换成浏览器能够解释的 CSS。Sass 和 LESS 样式表语言都包含 CSS 预处理器，并且这些语言可以添加变量、嵌套和 mixin 等功能。
  9. Redis 哈希 一个字符串和值的映射，用来表示 Redis 数据库中的对象。
  10. Redis 数据库 内存数据库，可以作为缓冲区和消息代理使用。常用在 Web 程序中存储用户会话和消息推送。

第 7 章

  1. 有意义的空格 JavaScript 用大括号、分号和换行来分隔语句。如果需要一个新的词法块，则用函数或控制语句。而在某些语言中，空格是有意义的，比如 Pug 中的每行代码都会用不同数量的空格形成缩进，从而形成代码块。
  2. mixin 通常是指一个类，这个类中定义了用在其他类中的方法。在 Sass 中，mixin 是 CSS声明的分组，可以在多处重用；在 Pug 中，mixin 用来定义可重用模板片段。
  3. 区块 lambda 因为 lambda 是匿名函数，所以 Hogan 中的区块 lambda 是将函数与模板中的标签关联起来的一种办法。
  4. XSS（跨站脚本）攻击 如果 Web 程序接受来自表单或 URL 参数的用户输入，并且那些值会重新出现在模板中，则有可能会被注入恶意代码。为了避免遭受这种攻击，必须先将收到的值进行转义。
  5. 子模板（partial） 小型的可重用模板。
  6. 词法作用域 变量的可见性是由它的作用域决定的。在 JavaScript 中，添加一个函数会增加新的作用域层级。那个函数中定义的所有变量对该函数中定义的所有函数来说都是可见的。

第 8 章

  1. ACID（Atomicity、Consistency、Isolation、Durability，原子性、一致性、隔离性和耐用性） 数据库要想满足 ACID，其操作必须是原子性的（操作或者成功，或者失败，但失败后数据库必须保持原样）、一致性的（数据只能以允许的方式改变）、隔离性的（确保能够并发执行）和耐用性的（变化发生后，即便经历了系统崩溃或重启，也必须保留下来）。
  2. Web worker 允许 JavaScript 运行在浏览器后台线程上的办法。
  3. BSON MongoDB 中表示对象的二进制格式。对象是由一组排好序的元素组成的。元素是由域名、类型、值构成的。BSON 支持的类型包括字符串、整型、日期和 JavaScript 代码。
  4. 面向文档的数据库 存储半结构化数据的数据库，这些数据没有预先定义好的模式，有时是 JSON 或 XML。比如 MongoDB 和 CouchDB。
  5. 发布/订阅 一种能够将消息发送给多个接收者的模式。
  6. 分布式数据库 存储在多台计算机上的数据库，虽然不一定，但有可能分布在不同的地理位置上。
  7. 复制集 一组 MongoDB 进程，可以让数据集保持一致。
  8. NoSQL 不用关系型数据库中那种表格关系的数据库。
  9. 关系代数 关系型数据库的理论基础，对所存储的数据和在其上执行的查询进行建模的依据。
  10. 缓存记忆（memoize） 一种优化技术，用于将函数的结果保存起来，这样就不用再次调用它了。
  11. 主键 数据库表中用来唯一标识每一行记录的那一列。
  12. 查询构建器 为程序员提供便利，不用再手动编写 SQL 的 API。
  13. 抽象漏洞 尝试将底层实现的大量细节和问题隐藏起来，以降低复杂度。
  14. 守护进程 在后台运行的程序，通常是在系统启动时自动启动的。
  15. 数据库模式 数据库中数据及其相互关系的正式定义。它是数据库的设计。
  16. 数据库事务 按照 ACID 属性组合在一起的一个或多个数据库操作。

第 9 章

  1. BDD（行为驱动开发） TDD 的扩展，其用不同的 API 风格来鼓励将注意力放在流程中的测试点上，测试什么，不测试什么，以及一次做多少测试。同时尽量改善测试失败提示及单元测试名称的可理解性。
  2. 模拟对象（mock） 行为像真正的对应物的值或对象，但通常要简单得多，只是为了测试模拟了刚好够用的行为。所以在测试中一般不会访问真正的文件或网络，因为速度可能会比较慢，如果进行破坏性操作的话，还会有危险，模拟物可以安全地模拟这些行为。
  3. 单元测试 在小的测试集（单元）中孤立测试模块的一小部分，比如函数或类的方法。
  4. 断言 确保表达式的计算结果符合期望。可以是一个简单的布尔语句、等式、或其他任何东西。在 Node 中，断言失败时会抛出异常。测试运行器可以捕获这些异常，并进行汇总以形成测试报告。
  5. typeof JavaScript 操作符，其可以根据指定对象或值返回一个字符串。
  6. 功能测试 测试整个系统中的某个功能。在 Web 开发中，这意味着要同时测试浏览器和服务器端，是全栈测试。
  7. 测试运行器 管理测试加载、执行和结果收集以便显示的程序。Mocha 就是测试运行器。
  8. 测试驱动开发（TDD） 先写测试，再写要进行测试的代码。

第 10 章

  1. Elastic Beanstalk 亚马逊提供的协调性服务，用于向它的其他服务，比如 EC2，做脚本部署。
  2. 亚马逊 EC2（亚马逊弹性计算云） 亚马逊的虚拟计算机服务。
  3. Docker 映像 Docker 用来创建容器的文件系统的映像。
  4. dyno Heroku 对自己的容器的叫法。用来运行服务器以及 Heroku 的服务器上独立环境中的所有命令。
  5. 内容交付网络（CDN） 交付静态内容的分布式服务器。
  6. sudo 在需要使用其他用户权限运行程序时使用的命令。一般在需要特殊权限时使用，比如编辑系统配置文件。
  7. SSH（安全 shell） 提供一个连接远程计算机的加密命令行（或 X11）界面。在 Web 开发人员刚开始配置新的服务器，或连接到服务器运行维护或调试命令时使用。
  8. 容器 一种虚拟技术，是为用户隔离出来的操作系统实例，运行在主操作系统之上。容器提供了额外的资源使用控制，有安全优势，并且可以快速搭建和销毁。
  9. 日志轮转 定期运行的命令，根据日期重命名日志文件，可能还会进行压缩以节省存储空间。

第 11 章

  1. 退出状态码 程序结束时返回的值。非零值说明有错误发生。
  2. 国际开放标准组织（Open Group） 发布了单一 UNIX 规范（Single UNIX Specification）的国际性非营利组织，该规范是一组标准，用来认证可以使用 UNIX 商标的操作系统厂商。
  3. 进程间通信 操作系统提供的程序间相互通信的方法。比如管道就是用一个程序的输出作为另一个程序的输入。甚至连文件都可以被当作进程间通信的一种方式。
  4. 参数 程序的参数是在命令行中提供的标志，用来指明启用或禁用某些功能。
  5. stderr 用于运行中程序输出错误信息的流。
  6. stdout 用于程序要显示的信息的输出流。
  7. stdin 运行中程序的输入流。
  8. 重定向 捕获一个程序的输出并将其作为输入发送给其他程序或文件。
  9. shell 能够输入命令和查看结果的命令行用户界面。之所以称为 shell，是因为它是包裹在操作系统外面的一层。

第 12 章

  1. Electron 渲染进程 Chromium Web 视图。
  2. Electron 主进程 管理 Electron app 并负责访问文件和网络的 Node 进程。
  3. 原生 用操作系统自带的 API 写成的程序或库。
  4. JSX React 程序使用混合了 JavaScript 的 HTML 片段。在浏览器中运行前会预处理为纯粹的 JavaScript。这种语言称为 JSX。
  5. Chromium 一个开源的浏览器，Google Chrome 浏览器的代码源自该项目。
  6. React Facebook 为搭建数据驱动的 Web 和移动端用户界面提供的库。
  7. 网络抓取 将 HTML 转换成结构化数据以便存储在文件或数据库中。
  8. 微格式 人类和软件都能解读的在 HTML 中包含结构化数据的方式。因为 HTML 有时不能清晰地表示结构化数据，所以可以在不借助任何特殊标签的情况下用微格式在 HTML中嵌入地址、地理信息位置、日历条目等数据。
  9. DOM（文档对象模型） 这个标准为 JavaScript 处理 HTML 定义了 API。DOM 是与语言无关的 HTML 处理接口。
  10. 构造器 创建并初始化 JavaScript 对象的函数。
  11. XPath 用来从 XML 文档中选取节点的查询语句。
  12. CSV（逗号分隔的值） 表格化数据的文本格式，一般用于数据库或电子表格程序。其中的值会用逗号分隔成列，用换行分行。
  13. 正则表达式 匹配字符串中的模式的表达式。
  14. 垂直搜索引擎 专注于特定范围的搜索引擎。
  15. robots.txt 网站用来告诉网络爬虫和抓取器什么内容可以扫描或什么内容不能抓取的标准。
